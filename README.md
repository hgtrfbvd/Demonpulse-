## DEPLOY ON RENDER

1. Push to GitHub
2. New Web Service → connect repo
3. Build: `pip install -r requirements.txt`
4. Start: `gunicorn app:app --bind 0.0.0.0:$PORT --workers 2 --timeout 120 --preload`
5. Environment variables:
   - `CLAUDE_API_KEY` — Anthropic API key
   - `SUPABASE_URL` — Supabase project URL
   - `SUPABASE_KEY` — Supabase anon key
   - `JWT_SECRET` — random secret (auto-generated by render.yaml)
   - `FLASK_SECRET` — Flask session secret
   - `ADMIN_PASSWORD` — override default admin password

## SIGNAL SYSTEM

Each race assigned exactly ONE signal:

| Signal | Meaning | Alert |
|--------|---------|-------|
| SNIPER | High confidence, low risk | 🔥 HOT (popup) |
| VALUE | Positive EV edge | HIGH |
| GEM | Hidden upside, market wrong on fav | MEDIUM |
| WATCH | Monitor, not yet ready | LOW |
| RISK | Warning flags present | NONE |
| NO_BET | Pass this race | NONE |

## API ENDPOINTS

### Auth
- `POST /api/auth/login` — returns JWT token
- `POST /api/auth/logout`
- `GET /api/auth/me`

### Races
- `GET /api/races/upcoming` — signal-enriched race board
- `GET /api/races/<uid>/analysis` — full race + runners + scored + signal

### Signals
- `POST /api/signals/generate` — generate signal from scored data
- `GET /api/signals/<uid>` — get saved signal

### Exotics
- `POST /api/exotics/calculate` — calculate any exotic type
- `POST /api/exotics/suggest` — auto-suggestions based on signal

### Betting
- `POST /api/bet/log` — place a bet (paper mode)
- `POST /api/bet/settle` — settle with WIN/PLACE/LOSS
- `GET /api/bets` — all bets for current user

### Admin
- `GET /api/audit/log` — full audit trail
- `GET/POST /api/users` — user management
- `POST /api/users/create`
- `PATCH /api/users/<id>`
- `POST /api/learning/run` — batch training
- `POST /api/sweep` — manual data sweep
- `POST /api/cache/clear`

### System (V7 kept)
- `POST /api/chat` — Claude chat pipeline
- `GET/POST /api/state` — system state
- `POST /api/bankroll/set`
- `GET /api/session/pl`
- `POST /api/simulator/run` — Monte Carlo simulation
- `GET /api/performance/chart`
- `GET /api/export/csv`

## SECURITY

- JWT tokens (HMAC-SHA256, no external lib)
- PBKDF2-SHA256 password hashing (260k iterations)
- Rate limiting: 10 login attempts / 5 min per IP
- Session timeout: configurable via SESSION_TIMEOUT_MIN
- All admin endpoints require `role=admin` check
- Audit log on all sensitive operations
- Secure HttpOnly cookies
